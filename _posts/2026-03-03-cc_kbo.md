# KBO 판타지 드래프트 웹 애플리케이션 - 프로젝트 요약

## 프로젝트 개요

KBO(한국야구위원회) 야구 기록을 기반으로 한 판타지 드래프트 웹 애플리케이션입니다. 4~12명의 참가자가 Snake Draft 방식으로 선수를 선택하고, 헤드 투 헤드(H2H) 또는 로티서리(Roto) 방식으로 경쟁하는 시스템입니다.

## 주요 기능

### 1. 사용자 인증

- JWT 기반 토큰 인증
- bcrypt 비밀번호 해싱
- 회원가입, 로그인, 사용자 정보 조회

### 2. 리그 관리

- 리그 생성 (4~12명, 봇 지원)
- 리그 타입 선택: H2H(헤드투헤드) / Roto(로티서리)
- 리그 가입 (비밀번호 기반)
- 리그 검색 및 멤버 관리
- **메인 리그 설정** (localStorage 기반, 전 페이지 공유)
- 리그 1개일 경우 자동 메인 설정

### 3. 드래프트 시스템

- **Snake Draft 방식**: 1라운드(1→N), 2라운드(N→1), 3라운드(1→N)...
- **총 26라운드**: 각 팀당 26명의 선수 드래프트
- **실시간 상태 조회**: 현재 픽 순서, 드래프트 진행 상황
- **선수 검색 및 필터링**: 포지션, 팀별 필터링
- **드래프트 결과 테이블**: 리그 페이지에서 완료된 스네이크 드래프트 결과를 표로 표시 (짝수 라운드 하이라이트)

### 4. 팀 관리

- 로스터 관리: 선수 명단 조회
- 라인업 설정: 활성/벤치 선수 설정
- 포지션 슬롯 관리

### 5. 헤드 투 헤드 매치업 시스템

- **주간 매치업**: 매주 팀 간 대결
- **10개 스탯 카테고리**: 타자 5개, 투수 5개
- **카테고리별 승패 판정**: 각 카테고리에서 우수한 팀이 1승
- **시즌 순위**: 매치업 승-패 누적 기록

### 6. 로티서리(Roto) 시스템

- **10개 카테고리 누적 순위 기반** 채점
- 각 카테고리별 순위 점수 합산으로 전체 순위 결정
- **대시보드에서 누적 시즌 순위 표시** (로토 리그 전용)
- 순위 변동(▲▼) 표시
- 상세 규칙: `docs/roto-scoring-rules.md` 참고

### 7. 트레이드 시스템

- 팀 간 선수 트레이드 제안/수락/거절
- 트레이드 이력 관리

### 8. FA (자유계약선수) 시스템

- **선수 방출**: 팀 로스터에서 즉시 삭제
- **FA 픽**: 스네이크 드래프트 방식으로 FA 선수 선택
- **픽 순서**: 누적 로티서리 역순 (Week 1은 드래프트 순서 역순 fallback)
- **실시간 드래프트**: 3초 폴링으로 턴/상태 업데이트
- **주차별 관리**: 주차 선택기로 과거/현재 FA 현황 조회

### 9. 선수 통계

- 선수 검색
- 상세 통계 조회
- 일별 기록 조회
- 시즌 집계 통계
- 선수 사진 지원

### 10. 자동화 시스템

- **매일 새벽 3시**: 전날 KBO 경기 데이터 크롤링
- **매주 월요일 00:01**: 주간 매치업 자동 생성
- **매주 월요일 00:30**: 지난주 매치업 결과 계산

#### 스탯 카테고리

**타자 (5개)**

1. OBP (출루율) - 높을수록 좋음
2. TB (총루타) - 높을수록 좋음
3. RBI (타점) - 높을수록 좋음
4. R (득점) - 높을수록 좋음
5. HR (홈런) - 높을수록 좋음

**투수 (5개)**

1. W+HLD+SV (승리+홀드+세이브) - 높을수록 좋음
2. IP (이닝) - 높을수록 좋음
3. ERA (평균자책점) - 낮을수록 좋음
4. K (탈삼진) - 높을수록 좋음
5. HR (피홈런) - 낮을수록 좋음

## 기술 스택

### Backend

- **Framework**: Flask 3.0.0
- **Database**: SQLite (로컬/서버) + Oracle Autonomous DB (데이터 저장)
- **Authentication**: PyJWT, bcrypt
- **Scheduler**: APScheduler
- **HTTP Client**: requests
- **Data Processing**: pandas, openpyxl
- **WSGI**: Gunicorn

### Frontend

- **HTML/CSS/JavaScript** (Vanilla)
- 반응형 모바일 대응 (hamburger nav)
- **CORS**: Flask-CORS

### 인프라

- **서버**: Oracle Cloud Compute Instance (Ubuntu)
- **프로세스 관리**: systemd (gunicorn)
- **배포**: SCP 기반 수동 배포

### 크롤링

- KBO 공식 API
- Naver Sports API

## 데이터베이스 구조

### 핵심 테이블 (12개)

1. **users**: 사용자 정보
2. **leagues**: 리그 정보 (H2H/Roto 타입 지원)
3. **league_members**: 리그 멤버 (봇 지원)
4. **players**: 선수 정보 (사진 URL 포함)
5. **player_stats**: 선수 일별 기록
6. **draft_picks**: 드래프트 픽 기록
7. **team_rosters**: 팀 로스터
8. **matchups**: 주간 매치업
9. **weekly_team_stats**: 주간 팀 통계 (집계)
10. **season_standings**: 시즌 순위
11. **fa_releases**: FA 방출 기록
12. **fa_picks**: FA 픽 기록

### 듀얼 DB 구조

- **SQLite**: 앱 서빙용 (로컬 개발 + 서버)
- **Oracle**: 데이터 영구 저장용
- 스키마: `schema.sql` (SQLite) / `schema_oracle.sql` (Oracle - 시퀀스/트리거 포함)
